# Portfolio Website Development Plan

## Objective
Build a modern, data-analyst-focused portfolio website for Arm Kantawee hosted on GitHub Pages.

## Design Requirements
- **Theme**: Modern dark mode.
- **Typography**: JetBrains Mono (via Google Fonts).
- **Layout**: Clean, responsive, single-page scrollable layout.

## Content & Sections
1. **Hero Section**:
   - Name: "Arm Kantawee"
   - Headline/Subline: Data Analyst
2. **Projects Section**:
   - Showcase 3 featured data analysis projects.
   - Include: Title, short description, and links to code/dashboard.
3. **Contact Details**:
   - Links to email, LinkedIn, and GitHub.

## Files Structure
- `/index.html`: Main site structure.
- `/style.css`: Dark theme styling, font imports, layout rules.
- `/script.js`: (Optional) Interactivity (e.g., smooth scroll).
- `/assets/`: Images or data visual previews.

## Implementation Steps
1. **Setup**: Initialize repository and basic folder structure.
2. **Style**: Define CSS variables for dark theme and typography.
3. **Skeleton**: Create HTML semantic markup for sections.
4. **Detail**: Populate project content and contact links.
5. **Deploy**: Follow GitHub Pages deployment steps.

## Verification
- Accessibility check (contrast).
- Responsive testing (mobile vs. desktop).
- Link verification.
